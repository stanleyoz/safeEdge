SafeEdge — System Architecture

Formally-verified edge safety agent (Hackathon Track 5: EdgeAgent). A real-time Signal-Temporal-Logic monitor runs autonomously on a Jetson Orin NX, while a Qwen-powered cloud brain deployed on Alibaba Cloud adds multimodal incident reporting, adaptive safety policy, and predictive risk forecasting.

The edge loop is safety-critical and never blocks on the cloud — if the network or cloud is unavailable, the Jetson keeps protecting people. The cloud adds intelligence when available.

2. Safety event → Qwen incident report

  1. The USB camera streams a frame; homography projects detections to ground-plane metres, and the edge pipeline detects, tracks, and extracts metric signals.
  2. STL Monitor computes robustness ρ for the spec “keep pedestrians safe from moving vehicles”.
  3. When ρ < 0 (specification violated), the Intervention Engine raises a WARNING/EMERGENCY event.
  4. Cloud Client sends POST /api/events with the event + JPEG frame — non-blocking.
  5. The backend passes image + structured STL data to qwen-vl-max.
  6. Qwen returns a natural-language incident report; the backend persists it to Tablestore.
  7. The dashboard receives the new incident over WebSocket. Meanwhile the edge keeps running at 30 Hz regardless of cloud latency.

3. Adaptive policy loop (closed-loop, edge ⇄ cloud)

  1. Every ~5 min the edge sends POST /api/policy/evaluate with the ρ-summary, recent event counts, and current STL parameters.
  2. The Policy Manager asks qwen-max to balance safety against false-alarm rate.
  3. Qwen returns a JSON patch, e.g. { "phi2": { "proximity_zone": 5.5 }, "phi4": { "stop_window": 65 } }.
  4. The backend returns the patch; the edge applies it via apply_cloud_params(), hot-swapping the formal safety thresholds live.

4. Why this architecture

Real-time safety STL robustness is computed on-device at 30 Hz — deterministic and formally grounded. No LLM sits in the safety-critical path.
Autonomy / resilience cloud_client.py is fire-and-forget with hard timeouts on a worker thread; cloud outages never stall the safety loop.
Sophisticated Qwen use Three distinct custom skills — multimodal reporting (qwen-vl-max) plus reasoning (qwen-max) for policy & forecasting — returning structured JSON applied back to the formal monitor.
Alibaba Cloud backend FastAPI on Function Compute (serverless, scale-to-zero) + Tablestore datastore — the deployed backend the hackathon requires.
Domain-portable Subjects/objects, signals, STL specs, and prompts are config-driven; the same pipeline retargets to other vision-safety domains (e.g. warehouse forklift–pedestrian).
